# 戒烟 App（QuitBuddy）V1.0 需求文档

## 1. 文档目的与范围

- 明确产品目标、功能边界、数据与技术约束，指导设计、开发、测试与上架。
- 覆盖 iOS 端（iPhone），包含小组件；Apple Watch 为可选增强；无后端（默认离线，本地数据为主，CloudKit 仅做可选同步）。

## 2. 背景与目标

### 用户价值

- 帮助用户制定戒烟计划、识别触发因素、用微干预度过渴求、可视化收益并保持连续性。

### 业务目标

- D1 留存 ≥ 45%，D7 留存 ≥ 20%（目标）。
- 用户连续无烟天数中位数 ≥ 7 天。
- App Store 评分 ≥ 4.7。

### 非目标（V1 不做）

- 医疗诊断/处方建议。
- 社区/社交动态。
- 复杂的云端账户系统。
- 第三方广告/追踪。

## 3. 术语

- **渴求事件**：用户出现想吸烟的时刻与上下文记录。
- **微干预**：呼吸练习、延迟计时、替代行为提示等快速行为工具。
- **里程碑**：3/7/14/30/90/180/365 天等关键达成节点。

## 4. 目标用户与典型场景

### 目标用户

- **新手戒烟者**：首次尝试，需强引导与高反馈密度。
- **反复者**：多次复吸，重启门槛要低，避免羞辱性文案。
- **数据驱动者**：关注钱/时间/健康收益与趋势。

### 典型场景

- 起床后渴求、社交饮酒、通勤/开车、工作压力、咖啡时段、独处/无聊时段等。

## 5. 关键用户旅程（高层流程）

1. 首启 Onboarding → 选择完全戒/逐步减少 → 设定起始日、每日基线、单价、每包支数 → 开启通知/小组件（可选）。
2. 日常使用：在仪表盘查看无烟天数/节省金额/趋势 → 一键记录渴求 → 触发“延迟 3 分钟/呼吸 60–120 秒”等 → 成就反馈。
3. 高风险时段：历史统计找出峰值时段，提前 10–15 分钟发提醒。
4. 复吸处理：温和记录并重启；连续计数重置，但累计节省与历史保留。

## 6. 功能性需求（FRD）

### 6.1 Onboarding（首次设置）

- 选择模式：完全戒 / 逐步减少（V1 仅展示，不做复杂计划引擎）。
- 输入：起始日期、每日基线支数、每包支数、价格（币种自由文本）。
- 可选：开启通知（时间段预设：9:00 / 14:00 / 21:00）。
- 结果：落库 `QuitPlan`；跳转仪表盘。

**验收要点**：

- 未填必填项不可完成。
- 完成后 1 秒内能进仪表盘。
- 返回后可在设置页修改。

### 6.2 仪表盘

- 展示：无烟天数、节省金额、避免香烟数（实时/分钟级刷新）。
- 下一里程碑卡片：显示目标天数与预计达成时间。
- 快捷入口：记录渴求、打开干预工具。
- （可选）近 7 天渴求强度火花线/柱图（V1 可用简单文本概览替代）。

**验收要点**：

- 指标计算与设置一致。
- 刷新不低于 60s 周期。
- 空状态有引导文案。

### 6.3 渴求记录与日志

- 字段：强度 1–10、触发情景（咖啡/酒精/压力/社交/驾驶/独处/其他）、是否吸烟、备注。
- 保存后即时反馈（如“👏 已坚持 3 次不吸！”）。
- 历史列表按时间倒序；可搜索/过滤（触发情景、是否吸）。

**验收要点**：

- 表单 3 步内完成。
- 失败有本地重试。
- 列表 1,000 条内滚动流畅。

### 6.4 干预工具

- 呼吸练习：圆形呼吸动画，节律 4-4-4-4（吸-停-呼-停），60–120 秒。
- 延迟计时：3 分钟倒计时，可暂停/继续。
- 替代建议卡（V1 简单文案库）：喝水、散步 2 分钟、嚼口香糖等。

**验收要点**：

- 后台切前台不中断（延迟计时按系统行为）。
- 完成后返回上一层并提示鼓励语。

### 6.5 通知（本地）

- 授权请求；固定提醒（用户自选时段）。
- 高风险提醒：近 7 天事件时间段统计，峰值前 10–15 分钟通知。
- 里程碑祝贺：达成当日推送（避免多条叠加）。

**验收要点**：

- 拒绝授权时提示轻量引导。
- 重复提醒 ID 去重。
- 冷启动后策略不丢失。

### 6.6 成就与勋章

- 固定节点：3/7/14/30/90/180/365 天；“连续 7 日未吸”。
- 成就弹窗 + 卡片页；可在设置中导出证书 PNG（V1 可留为占位）。

### 6.7 数据导出

- CSV：渴求事件（时间/强度/触发/是否吸/备注），UTF-8，按月或全量。
- 本地生成并系统分享；无网可用。

### 6.8 设置页

- iCloud 同步开关（默认关；开时使用 CloudKit 容器）。
- App Group 共享（与 Widget 共享统计快照，默认开）。
- 主题：系统/浅色/深色；强调色选择器。
- 隐私：数据清除、数据导出、隐私说明。
- 通知：时段管理、测试通知按钮。

### 6.9 小组件（WidgetKit）

- 小号/中号：显示“无烟天数 + 节省金额”。
- 数据来源：App Group 内 `shared_snapshot.json`。
- 刷新策略：时间轴 30 分钟；App 前台更新时手动推送快照。

**验收要点**：

- 安装后 30 秒内展示占位 → 真实数据。
- App 修改统计后 10 秒内 Widget 反映。

### 6.10 Apple Watch（可选）

- Complication：无烟天数。
- App：一键开始呼吸练习/延迟计时，简易渴求记录。

### 6.11 快捷指令 / App Intents（可选）

- “记录一次渴求（强度 X，触发：咖啡）”。
- “启动 3 分钟延迟计时”。

## 7. 非功能性需求（NFR）

- 平台：iOS 16+（建议 17+），iPhone；适配英/简中。
- 性能：冷启动 ≤ 1.2s；主线程长任务 ≤ 16ms；列表 60fps。
- 可靠性：核心数据写入成功率 ≥ 99.9%，崩溃率 ≤ 0.3%。
- 离线：全部核心功能离线可用；网络仅用于 iCloud。
- 隐私与合规：无第三方追踪；可导出与清除数据；隐私营养标签 Health/Fitness（不共享）。
- 可访问性：VoiceOver 标签完整；动态字体（Content Size Category）；对比度达标。
- 本地化：zh-Hans / en，日期/货币格式随系统。

## 8. 数据设计

### 8.1 实体：QuitPlan

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| startDate | Date | 戒烟起始日 |
| mode | String | `coldTurkey`/`gradual` |
| dailyBaseline | Int16 | 过去每日支数 |
| pricePerPack | Double | 包价 |
| cigsPerPack | Int16 | 每包支数 |

### 8.2 实体：CravingEvent

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| timestamp | Date | 事件时间 |
| intensity | Int16 | 1–10 |
| trigger | String | 咖啡/酒精… |
| didSmoke | Bool | 是否吸烟 |
| note | String? | 备注 |

### 8.3 共享快照（App Group JSON）

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| days | Int | 无烟天数 |
| money | Double | 累计节省 |
| updatedAt | Date | 更新时间 |

### 8.4 核心计算

- 无烟天数：`today - startDate`（向下取整，复吸不清零累计天数，仅清连续计数—V1 可不区分）。
- 避免香烟：`dailyBaseline × 已过天数 - 实际吸烟记录数`。
- 节省金额：`避免香烟 ÷ cigsPerPack × pricePerPack`。
- 时间回收（显示用）：`避免香烟 × 8 分钟/支（保守值）`。

---

## 9. 通知策略

- 固定提醒：用户自选 1–3 个时段（默认 9/14/21 点）。
- 高风险提醒：统计近 7 天事件分布，选择 Top2 时间段，提前 10–15 分钟推送。
- 里程碑提醒：达成当日 9:30 发送；避免与固定提醒冲突（去重）。

---

## 10. 权限与隐私

- 仅使用通知权限；无位置/联系人/健康数据。
- 本地数据默认存储；iCloud 可选；数据导出/清除入口。
- 隐私政策与免责：声明仅为自我管理工具，非医疗建议。

---

## 11. 监控指标与成功标准（KPI）

- 使用：日活、D1/D7 留存、周渴求记录次数中位数、微干预启动率、复吸后重启率。
- 结果：连续无烟天数中位数、节省金额累计。
- 质量：崩溃率、主线程卡顿、通知点击率。
- 评分：App Store 平均评分与评论关键词。

---

## 12. 发布与运营

### 测试清单（关键）

- Onboarding 全路径；设置修改后回写。
- Fetch/统计准确；跨日滚动正确（时区：Australia/Sydney）。
- 通知授权/日程/高风险/里程碑。
- 导出 CSV；数据清除；iCloud 开/关切换。
- Widget 首次加载/刷新；App Group 文件权限。
- 主题切换、深色模式、动态字体、VoiceOver。

### 上架清单

- 标题/副标题/关键词；6.7" 与 6.1" 截图（模板已备）。
- 审核备注：不含医疗诊断，仅戒烟辅助。
- 隐私营养标签：不共享、无追踪。

---

## 13. 风险与依赖

- iCloud 同步：用户跨设备期望与冲突解决（V1 建议“最后写入优先”）。
- 通知到达率与打扰平衡：增加“安静时段”设置。
- 数据丢失防护：关键写入后落盘 & 容错提示。
- 不同国家包装/价格差异：UI 保持币种与价格自由输入。

---

## 14. 错误处理与边界

- 无计划数据：仪表盘显示空态并引导进入 Onboarding/设置。
- 未授权通知：设置页置顶黄色条提示。
- Widget 无数据：显示占位卡“打开 App 同步”。
- 存储/写文件失败：重试与用户提示（含诊断引导）。
- 时区变化：以 `Calendar.autoupdatingCurrent` 与系统时区计算。

---

## 15. 关键功能验收标准（摘录）

- **保存渴求**：Given 表单必填已填、When 点击保存、Then ≤150ms 提示成功并可在列表第一项看到记录。
- **高风险提醒**：Given 最近 7 天存在 2+ 次同一时段事件、When 到达前置 10–15 分钟、Then 必达 1 条提醒且不与固定提醒重复。
- **Widget 刷新**：Given App 刚更新了快照、When 回到桌面、Then ≤10s 内组件数值刷新。

## 16. 版本规划（不含日期）

- **V1.0**：Onboarding、仪表盘、渴求记录、微干预、通知、成就、导出、设置（iCloud 开关/主题）、Widget、App Group。
- **V1.1**：Charts 视图、安静时段、CSV 导出过滤、证书分享。

## 17. 开放问题

- 逐步减少模式是否需要“每日目标支数曲线”？（V1 暂留为占位）。
- 复吸计数是否区分“连续天数”与“累计天数”？（V1 先显示累计，连续可二期加入）。
- 是否需要“紧急联系人/求助热线”模块？（需区域合规评估）。

---

## 设计备选/拓展思路

- **行为科学**：把“延迟—分散注意—重评估”做成 3 步引导微流程；加入“自我承诺卡”与“诱惑清单”。
- **个性化**：强度/触发 → 生成个人“风险雷达”与时间热力图。
- **激励机制**：可选打赏或一次性解锁主题；不影响核心功能。
